# academic-paper-skills 融入方案（面向当前毕业论文）

来源：`https://github.com/lishix520/academic-paper-skills`

## 一、我已吸收的核心方法

将原仓库双技能流程改造成当前项目可执行版本：

1. **Strategist（策略）阶段**
- 平台与格式标准提炼（学院毕业论文模板优先）
- 文献与研究空白梳理（围绕智慧医疗、风险预警、RAG）
- 形成可写作的细粒度大纲（到三级标题 + 字数预算）

2. **Composer（写作）阶段**
- 按章节写作，逐章质量门控
- 每章进行“论证/文献/清晰度/结构/格式”5维评分
- 全文再做7维终评，未达标即修订

## 二、当前论文的执行规则（已启用）

## 1) 结构与字数预算
- 摘要：300-500字
- 第1章 绪论：1800-2200字
- 第2章 需求分析与总体设计：2000-2500字
- 第3章 关键技术与实现：2600-3200字
- 第4章 实验与结果分析：2200-2800字
- 第5章 总结与展望：1000-1400字
- 总体目标：12000字左右

## 2) 章节质量门（Chapter Gate）
每章完成后按5维打分（各4分，满分20）：
- 论证质量
- 文献支撑
- 表达清晰度
- 结构连贯性
- 模板/格式符合度

通过线：>=16/20。

## 3) 全文质量门（Final Gate）
按7维打分（各10分，满分70）：
- 整体论证
- 文献整合
- 清晰可读
- 创新与贡献表达
- 方法严谨性
- 结构组织
- 模板符合度

通过线：>=56/70。

## 三、当前落地动作

1. 已引入“先策略后写作”的流水：先出细纲再写章。
2. 已将实验章改为“可量化结果驱动”，必须含：
- 指标定义（Acc/Recall/F1/AUC）
- 基线对比表
- 消融实验表
- 误差/案例分析
3. 已将“模板符合度”纳入硬门控（不再只追求可编译）。
